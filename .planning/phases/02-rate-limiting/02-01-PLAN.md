  <files>package.json, src/lib/rate-limit.ts</files>
  <action>
1. Install dependencies:
   ```
   npm install @upstash/ratelimit @upstash/redis
   ```

2. Create `src/lib/rate-limit.ts` with:

   - **Lazy singleton Redis client:** `getRedis()` returns `Redis | null`. If `UPSTASH_REDIS_REST_URL` is not set, log `console.warn('[rate-limit] UPSTASH_REDIS_REST_URL not set -- rate limiting disabled')` and return `null`. This is the dev fallback per user decision.

   - **Pre-configured limiters** using factory functions (each returns `Ratelimit | null`):
     - `globalLimiter`: sliding window 100 req / 60s, prefix `rl:global`
     - `authLimiter`: sliding window 5 req / 60s, prefix `rl:auth`
     - `apiLimiter`: sliding window 10 req / 60s, prefix `rl:api`
     - `newsletterLimiter`: sliding window 5 req / 60s, prefix `rl:newsletter`

   - Each limiter should be lazily created (create on first `.get()` call, cache thereafter). Use a simple object with a `get()` method that caches the Ratelimit instance.

   - **`rateLimit()` helper function:**
     ```typescript
     export async function rateLimit(
       limiter: { get: () => Ratelimit | null },
       identifier: string
     ): Promise<{ limited: boolean; headers: Record<string, string> }>
     ```
     - If limiter.get() returns null, return `{ limited: false, headers: {} }` (fail-open)
     - Otherwise call `limiter.get().limit(identifier)`
     - Return rate limit headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`
     - On limit exceeded, add `Retry-After` header (seconds until reset, minimum 1)

   - **`getClientIp()` helper:**
     ```typescript
     export function getClientIp(request: NextRequest): string
     ```
     Returns `request.ip || request.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || '127.0.0.1'`

   - Use `import { Ratelimit } from '@upstash/ratelimit'` and `import { Redis } from '@upstash/redis'`
   - Import `NextRequest` from `next/server` for the getClientIp type
