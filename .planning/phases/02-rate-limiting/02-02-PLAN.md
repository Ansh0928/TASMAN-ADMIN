Rewrite `src/middleware.ts` to add global rate limiting (SEC-09) and CSRF protection (SEC-10) while preserving existing session-check logic.

**IMPORTANT: Middleware runs at the Edge.** Cannot import from `src/lib/rate-limit.ts` if it imports Node.js-only modules. Instead, instantiate Upstash directly in middleware (Edge-compatible). The `@upstash/ratelimit` and `@upstash/redis` packages are Edge Runtime compatible.

**Changes to middleware.ts:**

1. Make `middleware()` function `async`

2. **Add CSRF Origin check FIRST** (before rate limiting, before session check):
   - Define `STATE_CHANGING_METHODS = new Set(['POST', 'PUT', 'DELETE', 'PATCH'])`
   - Define `CSRF_EXEMPT_PATHS` array: `['/api/stripe/webhook', '/api/auth/callback']`
   - For state-changing methods on non-exempt paths:
     - Get `origin` header and `host` header
     - If `origin` exists, parse it with `new URL(origin).host` and compare to `host`
     - If mismatch, return `NextResponse.json({ message: 'Invalid request origin' }, { status: 403 })`
     - If no Origin header, allow through (same-origin requests may omit Origin)
     - Wrap URL parsing in try/catch -- malformed Origin returns 403
   - Exempt paths: check if pathname starts with any CSRF_EXEMPT_PATHS entry

3. **Add global rate limit** (after CSRF, before session check):
   - Create lazy singleton `getGlobalRateLimiter()` that returns `Ratelimit | null`
   - If `UPSTASH_REDIS_REST_URL` is not set, skip (dev fallback)
   - Use `Ratelimit.slidingWindow(100, '60 s')` with prefix `rl:global`
   - Get IP: `request.ip || request.headers.get('x-forwarded-for')?.split(',')[0]?.trim() || '127.0.0.1'`
   - If limited, return 429 with `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining` headers

4. **Add auth-tier rate limit for NextAuth paths** (SEC-07 partial):
   - Create lazy singleton `getAuthRateLimiter()` with sliding window 5/60s, prefix `rl:auth:mw`
   - Apply to paths matching `/api/auth/signin` or `/api/auth/callback/credentials`
   - This handles NextAuth endpoints that can't have per-route rate limiting (because they re-export handlers)
   - If limited, return 429 with appropriate headers

5. **Update matcher** to run on all routes (needed for global rate limit + CSRF):
   ```typescript
   export const config = {
     matcher: [
       '/((?!_next/static|_next/image|favicon\\.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp|ico)$).*)',
     ],
   };
   ```
   This matches everything except static assets.

6. **Preserve existing session check logic** -- keep the session cookie check and redirect logic, but only apply it to the paths that need it (admin, wholesale/prices, account). Use path checks within the middleware function rather than relying on the matcher.

7. **Unskip middleware tests** in `src/__tests__/middleware.test.ts`:
   - Remove `it.skip()` and replace with `it()` for all CSRF and global rate limit tests
   - Ensure all tests pass

```typescript
import { rateLimit, authLimiter, getClientIp } from '@/lib/rate-limit';
// or apiLimiter / newsletterLimiter depending on endpoint

export async function POST(request: NextRequest) {
  // Rate limit check
  const ip = getClientIp(request);
  const { limited, headers } = await rateLimit(authLimiter, ip);
  if (limited) {
    return NextResponse.json(
      { message: 'Too many requests. Please try again later.' },
      { status: 429, headers }
    );
  }
  // ... existing handler logic (unchanged)
}
```
